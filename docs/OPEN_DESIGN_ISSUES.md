# Open Design Issues

This file is the lightweight backlog for unresolved design or contract gaps.

## Current Status

There are currently no active design issues tracked for `relaynews.ai`.

## When To Use This File

Add an item here only when implementation uncovers a real unresolved issue such as:
- a conflict between architecture and executable schema
- a public API contract gap that blocks frontend or backend work
- a routing, rendering, or caching ambiguity that affects implementation
- a security boundary that is not yet specified well enough to build safely

## How To Record A New Issue

Each new issue should include:
- the current state
- the implementation risk or impact
- the follow-up decision needed
- the affected files or docs

Keep this file focused on active blockers or unresolved design questions.
Resolved items should be removed or summarized in commit history rather than kept as a
long-running historical log.
